# Core Axioms of SYF

**Layer**: SYF Core  
**Status**: Canonical · v0.1-docs

These axioms are **non-negotiable** and have the same authority as SyFF.

## The Six Axioms

1. **No governance** — No voting, no admin, no override, no multisig
2. **No oracle** — No external input required for core operation
3. **No feedback** — No loop from output back to input
4. **No human parameters** — No tunable knobs, no configuration
5. **No intent** — No goal, no optimization target, no purpose
6. **No recovery** — Failure is valid; no restart path, no rollback

## Validation Rule

If **any** axiom is violated, the system is **not** SYF Core.

## Rationale

These axioms exist because:

- Governance introduces human judgment → unpredictable
- Oracles introduce external dependencies → attackable
- Feedback loops introduce optimization → drift toward goals
- Human parameters introduce discretion → capture
- Intent introduces purpose → misalignment possible
- Recovery introduces second chances → moral hazard

## The Meta-Axiom

> **No axiom can be violated to "save" another.**

A system that breaks one rule to preserve another is not SYF-compliant.
